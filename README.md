# AirQualityProject

Реализация различных подходов к обработке и очистке данных о качестве воздуха на примере UCI Air Quality Dataset. Проект демонстрирует применение классического анализа данных (Pandas), ленивых вычислений (Pipeline) и объектно-ориентированных паттернов проектирования (Strategy).

## 📊 Описание датасета

В проекте используется **Air Quality Dataset**.
* **Источник:** [UCI Machine Learning Repository](https://archive.ics.uci.edu/ml/datasets/Air+Quality)
* **Контекст:** Данные содержат ответы газочувствительных металлооксидных датчиков, установленных в химическом анализаторе в загрязненной зоне на уровне дороги в итальянском городе.

### Основные проблемы данных
1. **Пропуски данных:** В оригинальном датасете отсутствующие значения маркируются числом `-200`. Требуется их локализация и заполнение.
2. **Аномалии и выбросы:** Наличие экстремальных пиков, вызванных сбоями оборудования или внешними факторами.

## 🛠️ Архитектура и паттерны проектирования

Для гибкой обработки пропусков в проекте реализован поведенческий паттерн **Стратегия (Strategy)**. 

Класс-контекст `FillGapStrategy` принимает в себя одну из конкретных стратегий, унаследованных от абстрактного интерфейса базовой стратегии, и инкапсулирует логику заполнения пропущенных значений (или удаления поврежденных строк):

* `FillPreviousStrategy` — заполнение пропуска предыдущим валидным значением.
* `FillMeanStrategy` — заполнение средним арифметическим по признаку.
* `FillMedianStrategy` — заполнение медианным значением.
* `DropRowStrategy` — удаление строки, если в ней доля пропусков выше заданной.

## 🚀 Запуск обработки

### 1. Обработка через Pandas
Сценарий использует библиотеку Pandas для анализа и очистки данных, на выходе формируется отчет, показывающий динамику загрязнения по месяцам в дневное время.
### 2. Обработка через пайплайн
Сценарий использует концепцию ленивых вычислений и потоковых конвейеров. Данные обрабатываются поочередно с помощью итераторов и генераторов, что позволяет эффективно работать с большими объемами данных.

### 3. Обработка через Стратегии
Сценарий использует поведенческий паттерн проектирования Стратегия в сочетании с механизмом кастомных исключений. Этот подход обеспечивает максимальную гибкость и изолированность бизнес-логики.
